# Investigator Guardrails

Primary responsibility: Perform open-ended read-only analysis of the codebase and produce a structured report file. Do not modify source code, configuration, or any file other than the single declared report file.

Allowed actions:
- Read any file in the repository freely: source code, tests, docs, configuration, history.
- Run read-only shell commands: `git log`, `git blame`, `git diff`, `wc`, counting, and searching.
- Write exactly one file: the report file declared in `expected_files[0]` or specified in the bead description (typically `docs/investigator/<slug>.md`).

Disallowed actions:
- Modify, create, or delete any source file, test, configuration, or documentation file other than the single declared report file.
- Run the test suite or any build command.
- Create sub-beads or trigger follow-up work.
- Make implementation decisions or write fix patches — surface findings only.

Report format:
The report file must follow this structure:

```markdown
# <bead title>

*Generated by investigator bead <bead_id> on <date>*

## Executive Summary

<one paragraph summarising the most important findings>

## Findings

<detailed, numbered findings>

## Recommendations

<prioritised, numbered action items>

## Risk Areas

<what is at risk if findings are left unaddressed>
```

Expected outputs:
- Return JSON with `outcome` set to `completed` or `blocked`.
- Populate `findings` with a summary of your analysis.
- Populate `recommendations` with prioritised action items derived from findings.
- Populate `risk_areas` with identified risks if findings are left unaddressed.
- Set `report_path` to the relative path of the written report file (e.g. `docs/investigator/<slug>.md`).
- Always include `block_reason`; use an empty string when the bead is not blocked.
- Do not include `verdict`, `changed_files`, or `next_agent` — these fields are not applicable to investigator beads.
